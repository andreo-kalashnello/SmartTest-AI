# Чеклист: функционал продукта (по исходному ТЗ)

Каждый пункт помечен **v1-test** (минимум для тестовой версии и сдачи доков) и/или **june-full** (полный продукт, ~июнь). Часть пунктов **v1-test** опциональна — если не успеваете, переносите на **june-full** и явно фиксируете в презентации демо.

---

## 1. Блок преподавателя

### 1.1 AI-генератор тестов

- [ ] **june-full** Импорт материалов: загрузка PDF, DOCX, TXT; вставка текста
- [ ] **june-full** Генерация по промпту (тема, количество вопросов, уровень класса)
- [ ] **june-full** «Умные» дистракторы (правдоподобные неверные варианты)
- [ ] **v1-test** *Опционально:* один упрощённый сценарий AI (только промпт или только текст) — если бэкенд готов; иначе только ручное создание в v1

### 1.2 Редактор тестов (manual)

- [ ] **v1-test** Добавление, удаление, редактирование вопросов и вариантов ответа
- [ ] **v1-test** Один тип вопроса в v1 достаточно: один правильный ответ
- [ ] **june-full** Несколько правильных ответов, тип «правда/ложь»
- [ ] **june-full** Вложения изображений к вопросу
- [ ] **june-full** Банк вопросов / избранные вопросы для повторного использования

### 1.3 Настройки тестирования

- [ ] **june-full** Общий лимит времени на тест и/или на вопрос
- [ ] **june-full** Античит: персональное перемешивание порядка вопросов и вариантов
- [ ] **june-full** Окно доступности теста (дата/время начала и окончания)
- [ ] **june-full** Настройка обратной связи: показывать ли ошибки и правильные ответы сразу после завершения
- [ ] **v1-test** *Минимум:* возможность завершить тест и увидеть результат (правила показа — простейшие, захардкоженные или один флаг)

### 1.4 Дистрибуция и офлайн

- [ ] **v1-test** Уникальный PIN (например 6 цифр) и/или прямая ссылка для входа ученика
- [ ] **june-full** QR-код для проектора
- [ ] **june-full** Генерация PDF: бланки для учеников + ключ с правильными ответами для преподавателя

### 1.5 Аналитика и журнал

- [ ] **june-full** Live-мониторинг: кто проходит тест и на каком вопросе
- [ ] **june-full** Средний балл класса, рейтинг учеников
- [ ] **june-full** Подсветка «самого сложного» вопроса (больше всего ошибок)
- [ ] **june-full** Экспорт в Excel/CSV
- [ ] **v1-test** *Минимум:* список попыток по тесту с баллом и ФИО (без live)

---

## 2. Блок ученика

### 2.1 Старт

- [ ] **v1-test** Вход без полноценной регистрации: PIN + ФИО (или ник по согласованию)
- [ ] **v1-test** Адаптивная вёрстка под телефон и десктоп

### 2.2 Интерфейс прохождения (Test Player)

- [ ] **v1-test** Чистый экран вопроса, прогресс («Вопрос k из n»)
- [ ] **june-full** Таймер с визуальным предупреждением при малом остатке времени
- [ ] **june-full** Пропуск вопроса и возврат позже (если разрешено настройками)

### 2.3 Результаты и обратная связь

- [ ] **v1-test** Подсчёт баллов после «Завершить»
- [ ] **june-full** AI-пояснения ошибок (премиум): короткое объяснение, почему верный вариант правильный

---

## 3. Дополнительно и масштаб

- [ ] **june-full** Режим «соревнование» (Kahoot-подобный): общий экран + смартфоны, очки за скорость
- [ ] **june-full** Организации (B2B): кабинеты школ/вузов, агрегированная статистика
- [ ] **june-full** AI-проверка открытых ответов (эссе) по критериям

---

## Сводка: что заложить в тестовую v1 по смыслу

| Область | v1-test | june-full |
|---------|---------|-----------|
| Создание контента | Ручной редактор, простой тип вопроса | AI-импорт, типы вопросов, медиа, банк |
| Запуск теста | PIN / ссылка | QR, окна времени, античит, таймеры |
| Прохождение | Player + результат | Пропуски, сложные таймеры, AI-объяснения |
| Отчётность | Список попыток и баллов | Live, аналитика, экспорт, PDF |

---

## Definition of Done — тестовая v1 (функционально)

1. Преподаватель может подготовить тест с вопросами с одним верным ответом и выдать ученикам PIN (или ссылку).
2. Ученик заходит без аккаунта, проходит тест, видит свой результат.
3. Преподаватель видит список прохождений с баллами (даже если без графиков и экспорта).
4. Сценарий стабильно демонстрируется на одном окружении (локально или на VPS).
