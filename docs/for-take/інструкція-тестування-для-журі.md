# SmartTest AI — доступ для журі та сценарій тестування

**Проєкт:** SmartTest AI — веб-платформа для створення та проведення навчальних тестів з AI-генерацією питань із навчального матеріалу.

**Термін перевірки:** до **24 травня 2026** (включно).

| **Протокол** | HTTP (без SSL; для конкурсу достатньо) |

Рекомендовані браузери: Chrome, Firefox, Edge (остання версія). Мобільна версія підтримується.

## 2. Ролі користувачів у системі

У прототипі реалізовано **дві ролі** (модель освітнього сервісу, а не маркетплейсу):

| Роль | Опис | Обліковий запис |
|------|------|-----------------|
| **Викладач** | Створює тести (вручну або через AI), отримує PIN, переглядає спроби учнів | Email + пароль |
| **Учень** | Проходить тест за PIN-кодом, **без реєстрації** | Логін не потрібен — лише PIN і ПІБ |

Окремих ролей «покупець / продавець / партнер» у проєкті немає — це платформа для викладача та учня.

## 4. Сценарій перевірки (≈15–20 хв)

### Крок A — Викладач: перегляд готового тесту

1. Відкрити http://smarttestai.duckdns.org/login  
2. Увійти з обліковими даними з п. 3.1.  
3. На дашборді знайти тест **«Демо: основи алгебри (для журі)»**.  
4. Відкрити **редагування** тесту — переконатися, що є питання з варіантами відповідей.  
5. Перейти в **«Спроби»** (attempts) — після проходження учнем тут з’явиться запис з балом.

### Крок B — Учень: проходження тесту

1. Відкрити **інкогніто** або інший браузер (щоб не змішувати сесію викладача).  
2. Перейти на http://smarttestai.duckdns.org/join  
3. Ввести PIN **`482916`** і ПІБ **`Тестовий Учень`**.  
4. Відповісти на питання, натиснути **«Завершити тест»**.  
5. Переглянути **результат** (бал / відсоток).

### Крок C — Викладач: AI-генерація нового тесту (ключова фіча)

1. Знову увійти як викладач.  
2. **«Новий тест»** → http://smarttestai.duckdns.org/dashboard/tests/new  
3. Заповнити, наприклад:  
   - **Тема:** «Лінійні рівняння»  
   - **Кількість питань:** 5  
   - **Складність:** середня  
   - **Мова:** українська  
   - **Матеріал:** короткий текст або завантажити PDF/зображення конспекту  
4. Натиснути **згенерувати через AI** (потрібен інтернет; генерація 30–90 с).  
5. Переглянути чернетку, за потреби відредагувати питання → **зберегти**.  
6. На дашборді скопіювати **новий PIN** і повторити крок B з іншим ПІБ.

> **Примітка:** якщо AI тимчасово недоступний (ліміт безкоштовної моделі), основний сценарій все одно можна перевірити через демо-тест (кроки A і B) та ручне редагування питань.

### Крок D — Перевірка аналітики

1. Як викладач відкрити сторінку спроб демо-тесту.  
2. Переконатися, що відображаються **ПІБ учня**, **бал**, **час**.
